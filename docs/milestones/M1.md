### 阶段 3：Rust 后端核心
- [ ] `core/platform.rs`：系统检测（OS、架构、Node.js 版本、OpenClaw 版本和状态）
  - 检测命令：`node -v`、`npm -v`、`openclaw --version`、`openclaw gateway status`
  - 检测端口 18789 是否监听（`lsof -i :18789`）
- [ ] `core/process_runner.rs`：进程执行 + 实时日志推送（Tauri emit `install-log` 事件）
- [ ] `core/config_manager.rs`：读写 `~/.openclaw/openclaw.json`
  - **重要**：写入前备份为 `.bak`，保留 `meta`、`wizard`、`gateway.auth.token` 不覆盖
  - 只更新以下字段：`agents.defaults.model.primary`、`channels.telegram.*`、`plugins.entries.*`
  - API Key 写入 `~/.openclaw/.env`（不写进 JSON）
- [ ] `commands/sysinfo.rs`：`get_system_info` IPC 命令
- [ ] `commands/installer.rs`：`run_install` IPC 命令（调用 shell 脚本）
- [ ] `commands/config.rs`：`save_config` IPC 命令

### 阶段 4：Shell 脚本准备
- [ ] 基于 `install-macos.sh`（openclaw-remote-deploy-template）改造，支持三种 Node.js 安装路线：
  1. 已有 Node.js 20+ → 直接跳过
  2. 有 Homebrew → `brew install node@22`
  3. 无 Homebrew → nvm 用户级安装（无需 sudo）
- [ ] 复制 `check-system.sh`、`check-openclaw.sh`（来自 openclaw-remote-deploy-template）
- [ ] 配置 `tauri.conf.json` bundle.resources 打包脚本
- [ ] 进度关键词匹配（用于进度条）：`Installing Node`、`npm install -g openclaw`、`gateway listening`

### 阶段 5：前端页面
- [ ] `pages/Welcome.tsx`：欢迎页 + 系统信息卡片
  - 显示：macOS 版本、架构、Node.js 版本、OpenClaw 状态（已安装/未安装）
  - 两个入口：**一键安装**（未安装）| **进入管理**（已安装）
- [ ] `pages/Installing.tsx`：安装进度 + 实时日志面板
  - 订阅 Tauri `install-log` 事件
  - 进度条通过关键词匹配推断百分比
- [ ] `pages/ConfigWizard.tsx`：分步配置向导（4步）
  - Step 1：AI 模型选择（→ `agents.defaults.model.primary`）
  - Step 2：API Key 输入（→ `~/.openclaw/.env`）
  - Step 3：Telegram Bot Token + 用户 ID（→ `channels.telegram.botToken` + `allowFrom`，可选跳过）
  - Step 4：完成确认
- [ ] `components/LogPanel.tsx`：实时滚动日志组件
- [ ] `components/StepProgress.tsx`：步骤进度指示器
- [ ] `App.tsx`：路由配置（Welcome → Installing → ConfigWizard → Dashboard）

### 阶段 6：集成测试

## 关键技术参考

### OpenClaw 运行命令
```bash
openclaw --version
openclaw status
openclaw gateway start / stop / restart / status
```

### 错误日志路径
`~/.openclaw/logs/gateway.err.log`

### 70% 故障根因
Plugin 路径必须用绝对路径，相对路径会导致 Gateway 无法启动。
